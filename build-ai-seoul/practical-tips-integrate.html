Tip 01

Use the Right Model: Start with Flash

Don't use a "sledgehammer" to crack a nut. Flash is the industry "sweet spot." Build on Flash and only "promote" failing tasks to Pro.

from google import genai

client = genai.Client()

# Start with Flash
response = client.models.generate_content(
    model="gemini-3-flash-preview",
    contents="Extract the data...",
)

# Only promote to Pro if needed
# model="gemini-3.1-pro-preview"

Tip 04

Use Markdown Format w/ Sections

Gemini is trained on structured data. Using Markdown helps distinguish between instructions, context, and tasks.

# Role
You are a helpful assistant.

# Constraints
1. Be objective.
2. Cite sources.

# Context
...

# Task
[Insert the specific user request here]
Tip 05

Put Instructions at the End of Long Context

"Recency bias" is real. Place your PDF or codebase first, and your question at the very bottom.

from google import genai

client = genai.Client()

sample_file = client.files.upload(file="large_file.pdf")

response = client.models.generate_content(
    model="gemini-3-flash-preview",
    contents=[
        "Here is a PDF document about <topic>:",
        sample_file,
        "Summarize this document",
    ],
)

print(response.text)
Tip 06

Give a Map, Not a Manual

Keep instructions short. Use a "Table of Contents" that points the agent to deeper sources of truth.

You are an autonomous developer. Follow the repository map.
Do not guess—read the corresponding doc first.

├── ARCHITECTURE.md: How to structure new services and handle DI.

Tip 07

Enforce Strict Grounding

Add a "negative constraint" to prevent hallucination when working with provided context.

Answer ONLY using the provided text.
If not present, say 'unsupported'.
Do not use outside knowledge.

Tip 08

Clear JSON Schema Descriptions & Types

Use the description field in your schema for instructions. Use strong typing (integer, string, enum) whenever possible.

class SupportTicket(BaseModel):
    summary: str = Field(
        description="A concise summary of the user's technical issue."
    )
    category: Literal["billing", "technical", "other"] = Field(
        description="The bucket this issue falls into."
    )
    impact_score: int = Field(
        description="A severity score from 1 (low) to 10 (high)."
    )

Tip 10

Trust but Verify

Structured output guarantees valid JSON (syntax), not valid logic (semantics). Always validate values in your code.

def process_gemini_response(json_response):
    try:
        # STEP A: Syntactic Validation (Is it valid JSON?)
        update = InventoryUpdate.model_validate_json(json_response)

        # STEP B: Semantic Validation (Is the data correct?)
        if update.product_id not in VALID_PRODUCT_IDS:
            raise ValueError(f"Product ID {update.product_id} not in DB.")

        if update.new_stock_level < 0:
            raise ValueError("Stock level cannot be negative.")

        print(f"✅ Successfully updated Product {update.product_id}")

    except ValidationError as e:
        print(f"❌ Syntax Error: bad JSON structure: {e}")
    except ValueError as e:
        print(f"⚠️ Semantic Error: hallucinated data: {e}")

Tip 19

Build Agents with the Interactions API

The recommended way to build multi-turn, tool-using agents. Handles conversation state, tool execution loops, and streaming — so you don't have to.

from google import genai

client = genai.Client()

# First turn
interaction = client.interactions.create(
    model="gemini-3-flash-preview",
    input="What is the capital of France?"
)
print(interaction.outputs[-1].text)

# Second turn — server remembers context
interaction2 = client.interactions.create(
    model="gemini-3-flash-preview",
    input="What language do they speak there?",
    previous_interaction_id=interaction.id
)
print(interaction2.outputs[-1].text)
